# Monitor Service

Система моніторингу доступності доменів з підтримкою сповіщень через Email та Telegram.

Загальні відомості
* Backend: Laravel 13
* База даних: MySQL 8.4
* Фронтенд: Vue/Inertia.js/Tailwind CSS (Повністю згенерований AI)
* Запуск локальний запуск на Docker (пересклав sail)/Віддалений запуск Docker + nginx з https 
* Система контролю версій: GitHub з мінімальним CI/CD для деплою на хостинг (self-hosted runner)
* Хостинг VPS Kamatera/Домен nic.ua
* Сповіщення Email/Telegram бот


## Функціональність

- **Моніторинг доменів** — періодична перевірка доступності URL з налаштовуваним інтервалом
- **HTTP методи** — підтримка GET та HEAD запитів
- **Перевірка контенту** — можливість перевірки наявності очікуваного тексту у відповіді
- **Гнучкі сповіщення** — Email, Telegram або обидва канали одночасно
- **Історія перевірок** — зберігання результатів з автоматичним очищенням старих даних
- **Dashboard** — візуалізація статусу всіх доменів користувача
- **Horizon** — візуалізація статусу Jobs для перевірок

## Локальний запуск

#### 1. Клонування репозиторію
```shell
git clone git@github.com:NoJustify/monitor-service.git && cd monitor-service
```
#### 2. Налаштування середовища
Створити .env з .env.example

#### 3. Побудова docker для першого запуску
```shell
chmod +x build.sh && ./build.sh
```
